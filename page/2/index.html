所以”提升产品价值“最大的思维倒向就是”以产品为中心“，”以终为始的数据闭环“为驱动，用精益思维（以最小的成本创造最大的价值）贯彻”以提升产品价值为目标“实施产品方案，确保产品实现最大化”收益“

什么是产品经理？

发表于2018-11-06更新于2020-09-19字数统计310阅读时长1分

这个话题很有争议性，对于刚入行的或者是想转行产品经理的，必须先了解对”产品”的定义。

产品这个词其实主要来源于互联网，那么我这里所指的产品就是互联网的产品。

我们可以先从一个概念开始，简单的去理解互联网产品。

“产品”这个词起初是从传统行业里边延伸而来的，在传统意义上它叫商品。

比如说，我们在日常里面所接触到的数码产品，包括我们吃的食物、喝的饮料、用的护肤品等等，这些都是叫做产品，也可以统称为商品。

但是在互联网领域里，产品它不单单指的是商品，它其实更多的时候代表是一种利用了it技术实现的一个功能，它是互联网企业，为了满足互联网用户需求，和用户欲望的一种无形载体。

简单的可以理解为，互联网”产品”指的是Web网站，或者APP应用为了满足用户需求而创建的用于运营的功能及服务，它是功能与服务的集成体。

线上教育瓶井该如何突破

发表于2018-11-02更新于2020-09-19字数统计3.1k阅读时长15分

I. 线上教育和线下教育有什么区别吗？线上教育的优势和劣势在哪方面？

答：这是很多人都在思考的问题，我有时候也在想线上教育和线下教育的区别到底是什么，我个人觉得最明显的区别是直观模式的不同，无论什么教育模式，比如一对一、小班、大班，线下都需要一个物理上的空间给学生进行知识传授，而线上需要的是虚拟空间，这形成了线上教育对线下教育的优势，也是多数线上教育公司营销时突出的点，即节省路上时间。第二个比较明显的区别则是课堂体验效果，我个人认为线上教育的课堂体验效果会逐渐趋近线下，但要想超越，需要新技术的成熟和颠覆，目前来说，线上教育的体验效果要弱于线下，这也是线下未被线上替代的主要原因，因为线下更有互动性，更容易产生老师与学员的亲密度，这一点将是未来线上教育主攻的方向。至于其他的区别，比如运营成本降低、教学效率提升等，我觉得都是在前两者基础上就可以得到结论的，就不在此赘述了。

II. 在线教育是否会替代传统教育行业？如果会代替线下教育，那么实现的形式和时间节点是怎样的？

问题描述：如果在线教育无法取代线下教育，那么在线教育可能以某种形式服务于真实课堂吗？

答：先说我的答案，线上教育永远不会替代线下教育，线上教育和线下教育将来一定是同时存在的，而且未来可能会出现线上教育、线下教育之外的另一种教育形式，具体是什么我说不好，目前来看，是以VR/AR为技术基础的可能性比较大。那么，线上教育在将来会以何种方式存在？现在有的线上教育模式，是在线1对1、在线小班课、在线双师大班，未来依旧会以这三种为主，只不过会有一些细节的改动，比如增加小班配辅导老师，双师大班加小组课等等。

III. 在专业教育方面，大量初级学生被线下教育机构截流，线上教育在这块有哪些突破思路呢？

问题描述：线上教育和线下教育的竞争越演越烈，在面临线下教育机构截流这个问题上，线上教育还有哪些突破思路呢？

答：这是一个很好的问题，也是从事教育的人很少关注的问题，其实我也是第一次被问到这种问题。我个人觉得，初级学员被线下机构截流是很正常的现象，主要原因有这样两个，第一是线上教育还难以覆盖到中等以下的学生，目前在线教育的模式多适合尖子生和自控力强的学生，这些学生不用家长监督，可大概率续费；第二是在线教育产品很少做本地化，多数是集中在一线城市，但即使是一线城市，区域间的教材版本都不一样，这必然促使一些学生选择线下机构。至于破局，无非两个思路，一是做本地化产品研发和招生运营，二是加大课堂模式的体验创新以及提高课后服务质量。

IV. 线上教育细分领域现在空间最大的是哪块？（比如成人、幼儿、出国留学等）

问题描述：线上教育现在竞争尤为激烈，请问还有哪些细分空间可以发展?（比如成人、幼儿、出国留学等）又如何发展？

答：目前在线教育的市场规模是4000亿左右，主要分四大块：幼教、k12、职教、留学。最大的一块无疑是k12，占45%以上，发展空间也最大，像腾讯、网易、头条、百度这些互联网巨头都往里下注圈地，传统巨头新东方在线和学而思网校也一直快速发展，还有一些题库起家的猿辅导、学霸君、作业帮，少儿外语及编程玩法，如VIPkid、核桃编程之类的，都是近年新冒出来的，说明这个领域是发展空间很大的，而其他领域由于市场较小，发展空间也有，但速度和规模都难以达到k12，个人看好这个领域。

V. 目前在线教育都集中在一二线城市，三四线市场怎么样？如何打开全国市场，布局又是怎样的呢？

答：渠道下沉是整个互联网的趋势，在线教育也是如此，虽然它较整个互联网的发展较晚一些，但趋势赶得很快。目前三四线城市有很大的教育需求，线下教育由于价格较高以及需要进行物理空间移动，在线教育很适合发展，但最大的问题就是对当地家长需求的把握如何精准，目前看到的两种布局方式比较好，也是较为主流的方式，一个是线下双师模式，这一点教培双子星（新东方和好未来）已经开始布局，可以跟踪模仿，另一个是体验店模式，这个目前做的有东方优播，无忧，朴新等，这两个模式可以研究一下。

VI. 线上教育现有的商业模式有哪些，还有创新空间吗？

问题描述：线上教育现在的盈利方式都是怎样的？除了老师在线授课的方式，还有什么其他的创新空间吗？

答：线上教育的商业模式，我个人分为四种，第一个在线教育平台，即邀请机构和老师入驻讲课，典型的是腾讯课堂，第二个是自营录播网校，课程以录播为主，早期的网校都是这种模式，第三是自营直播网校，比如在线1对1，在线小班，在线双师大班等，课程以直播形式呈现，老师与学生有互动，现在大多数在线教育玩家都使用这一模式，第四种是自营社群训练营，这是近两年出现的模式，主要得益于微信流量红利以及社群思维风靡，课程以录播短视频为主，学员需要打卡传播，典型玩家是宝宝玩英语。目前在线教育的模式没有大创新的可能，可能的方式是和线下结合，这是目前比较明朗的趋势。

VII. 在线教育怎么形成用户忠诚度，流失成本高吗？

问题描述：在线教育行业目前是怎么形成用户忠诚度的呢？今天我用这家的产品听课，明天我用他家的产品听课，怎么保证用户的流失呢？

答：在线教育产品的忠诚度，主要靠服务和课程质量来保证，这部分工作叫做续报招生和口碑招生。做好续报招生或口碑招生，需要考虑很多，最核心的就是两个，一个是保证老师的教学效果，另一个是保证与学生的良好关系，前者叫专业度，后者叫亲密度，这两个是底线，做好这两者的目标是超出用户预期。而关于专业度和亲密度怎么做，最主要的就两条，教学要标准化，收入与续报率挂钩。另外，教学产品体系也是很重要的一点，是不亚于专业度和亲密度的另一个核心，必须保证教学层级明确，教学目标清晰。

VIII. 在线教育如何解决教育过程缓慢的问题？

问题描述：对任何学习者来说，学会一个技能，都需要反复的训练和指导，见效非常缓慢，因为人脑中网络、髓鞘质等物理改变都需要时间，这是教育过程不可逾越的规律。在线教育有哪些手段可以有效解决学习效率问题？在线教育又如何替代讲师针对性指导的过程？

答：教育本身就是个慢过程，学生学习出现效果也是需要时间积累的，想要从根本上解决是不可能的，是违背规律的，但是却可以提高效率。在线教育在这个方面确实要比线下有优势，主要体现在灵活、碎片、及时。“灵活”就是不受空间限制和时间限制，尤其是录播模式，想什么时候看就什么时候看，随时随地学习；“碎片”就是可以不用花费一大段时间集中学习，而是可以分步骤的学习，像做任务一样去完成，这样比较符合学习规律；“及时”就是及时反馈，在互联网未大规模普及的时候，及时反馈其实很难做到，现在学员有问题可以直接联系到老师，老师也能及时联系到学员，沟通更高效，也更有安全感。

IX. 线上教育侵权、抄袭、付费变免费，版权问题怎么解决呢？怎么避免？

问题描述：由于财力有限，经藏买盗版课程。突然引发思考在线教育行业版权问题都是怎么解决的呢？又如何避免呢？

答：这一问题个人关注的不多，一旦涉及到版权问题，只有去打官司，这也是目前最好的办法，但会花费很多精力和财力，多数创业公司估计很难耗得起，而且也避免不了，个人觉得现在大家的版权意识越来越强，国家法律力度也越来越大，未来应该可以得到妥善解决。

X. 移动直播和在线教育的结合会不会是在线教育可发展的一个方向？

问题描述：在线教育和移动直播有可能很好的结合呢？会不会是未来教育可持续发展的一个点嘞？

答：移动直播目前在娱乐业发展比较好，主要模式是网红+打赏，而关于移动直播与在线教育的结合，个人觉得这个想法很好，但未必是一个发展方向，因为它和现有的直播模式没有大区别，甚至可以认为是一个东西，不过对于游学教育，这个是可以考虑的方向，但我觉得就整个在线教育来说，这个还不足以构成一个方向。

XI. 5G下的在线教育会不会是一个成为主流教育的契机？

问题描述：5G成为19年一个大的风口，未来5G下的在线教育是怎样的呢？会不会因此成为了一个登上主流教育的契机？

答：5G是下一个技术革命的重点之一，它意味着数据传输速度更快，这对在线教育来说是非常好的事情，当然也是在线教育成为主流教育的契机，原因有二，一是有极大可能创造逼近线下的线上体验效果，因为联网设备更丰富，尤其是和vr等结合可以创造虚拟现实环境，这将是革命性的变革，二是向偏远地区普及高质量教育成为可能，因为网络成本大大降低，偏远地区的孩子可以通过在线的方式更轻松享受到名师教育等资源，对社会有益，对在线教育成为主流更有帮助。

如果你善用者这六点，公司的扛把子非你莫属

发表于2017-11-02更新于2020-09-19字数统计707阅读时长3分

正如之前一篇文章我们所看到的其实编程并没有多难。比较难的部分实际上是先要得到一个程序。计算机是比较务实的，你必须先告诉它们要做什么，直至最后的细节。编写大型程序是一项非常艰巨的挑战，如果没有系统的方法实践，几乎是不可能完成的。

为了方便顺畅的编写程序我们通常会将创建程序的过程分为几个阶段，并依据每个阶段中产生的信息来进行分工工作。

1、分析问题

首先确定我们要解决的问题是什么。尝试竟可能的分析透彻（多进行了解）“知己知彼”，只有分析透彻才能知道要解决什么样的问题，否则一定不要开始去解决它。没搞清楚问题之前就去“实践”一定会给后续的开发带来很多问题。

2、确定规格说明

准确的描述程序将要怎么执行（程序将做什么）。此时，不必担心程序“怎么做”（代码怎么写），而是要确定它“做什么”。对于简单的程序，这包括仔细描述程序的输入和输出是什么以及他们的相互关系。

3、创建设计

这个步骤就是规划程序的总体结构。白话说就是描述程序怎么做的地方做成什么样子。主要任务是设计算法来满足规格说明。

4、实现设计

将设计翻译成计算机语言并放入计算机中来运行（简称编程）。在后面的文章中，我们将算法实现为 Python 程序。

5、测试/调试程序

试用你的程序，看看它是否按预期工作。如果有任何错误（通常称为“缺陷”），那么你应该回去修复它们。定位和修复错误的过程称为“调试”程序。在调试阶段，你的目标是找到错误，所以应该尝试你能想到的“打破”程序的一切可能。记住这句老格言：“没有什么能防住人犯傻，因为傻子太聪明了。”

6、维护程序

继续根据用户的需求开发该程序。大多数程序从来没有真正完成，它们在多年的使用中不断演进。

类和对象的关系理解：

对象是数据，对象本身不包含 方法。但是对象有一个指针指向一个类，这个类里可以有方法。

类和对象是不可分割的，有对象就必定有一个类和其对应，否则这个对象也就成 了没有亲人的孩子（但有一个特殊情况存在，就是由

标量进行强制类型转换的object,没有一个类和它对应。此时php中一个称为“孤儿”的stdClass类就会收留这个对象。

类的组合与继承

在一份代码清单里，定义了两个类，一个是person,一个是family;在family类中创建person类的对象，并把这个对象视为family类的

一个属性，并调用它的方法处理问题，这种复用方法叫做“组合”，还有一个复用方式，就是继承。

耦合：一个软件结构内不同模块之间互联程度的度量，也就是不同模块之间的依赖关系。

低耦合：指模块与模块之间，尽可能地各个模块独立；模块与模块之间的接口尽量少而简单。

解耦：要解除模块与模块之间的依赖。

组合和继承两者有很高的相似度，在两者皆可用的情况下，更倾向于使用组合。为啥呢？

继承破坏封装性

鸟类是父类，鸭子继承鸟类，鸭子岂不是会飞了？组合，从语义上来说，要优于继承。

继承是紧耦的

子类与父类捆绑在一起。组合仅通过唯一接口和外部进行通信，耦合度低于继承。

继承扩展复杂

多层继承和子类的增加，将设计大量方法重写。

不恰当地使用继承可能违反现实世界中的逻辑

组合并非没有缺点，在创建对象时，组合需要一一创建局部对象，这一定程度上增加了一些代码。

显然组合增加了代码量，组合还有其他缺点，不过总体来说，是优点大于缺点。

PHP5对面向对象的特性做了许多增强，其中就有一个SPL(标准PHP库）的尝试。SPL中定义了一些接口，其中最主要的就是

Iterator迭代器接口，通过实现这个接口，就能使对象能够用于foreach结构，从而在使用形式上比较统一。

实现Iterator的类，必须实现Iterator接口定义的五个方法current,key,next,rewind,valid

接口是对多重继承的一种变相实现，而在讲继承时，我们提到了用来实现混入氏的Traits,实际上，Traits可以被视为一种加强型的接口。

Traits和接口很像，Traits是可以被导入的（use语法），且已经实现了的方法列表。

反射

面向对象编程中对象被赋予了自省的能力，而这个自省的过程就是反射。

反射，直观理解就是根据到达地找到出发地和来源。比方说，我给你一个光秃秃的对象，我可以仅仅通过这个对象就能知道它所属的类，拥有哪些方法。

反射指在PHP运行状态中，扩展分析php程序，导出或提取出关于类、方法、属性、参数等的详细信息，包括注释。这种动态获取信息以及动态调用对象方法的功能称为反射API.

反射有什么用？

反射可以用于文档生成。因此可以用它对文件里的类进行扫描，逐个生成描述文档。

既然反射可以探知类的内部结构，那么是不是可以用它做Hook实现插件功能呢？或者是做动态代理呢？

<?php

class mysql {

function connect($db){

echo “连接到数据库${db[0]} \r\n”;

}

}

class sqlproxy{

private $target;

function __construct($tar){

$this->target[] = new $tar();

}

function __call($name,$args){

foreach($this->target as $obj){

//$obj是一个对象而不是类，但是这里也可以是ReflectionClass反射API

$r = new ReflectionClass($obj);

if($method = $r->getMethod($name)){

if($method ->isPublic() && !$method->isAbstract()){

echo “方法前拦截记录Log \r\n”;

$method->invoke($obj,$args);

echo “方法后拦截\r\n”;

}

}

}

}

}

$obj = new sqlproxy(‘mysql’);

$obj->connect(‘member’);

这里简单说明一下，真正的操作类是mysql类，但是sqlproxy类实现了根据动态传入参数，代替实际的类执行，并且在方法运行前后进行拦截（这就是TP中before，after方法或者laravel框架中中间件的实现原理），并且动态地改变类中的方法和属性。这就是简单的动态代理。

在平常开发中，用到反射的地方不多：一个是对对象进行调试，另一个是获取类的信息。在MVC和插件开发中，使用反射很常见，但是反射的消耗也大，在可找到替代方案的情况下，就不要滥用。

php有Token函数，可以通过这个机制实现一些反射功能。从简单灵活的角度讲，使用已经提供的反射API是可取的。

很多时候，善用反射能保持代码的优雅和简洁，但反射也会破坏类的封装性，因为反射可以使本不应该暴露的方法或属性暴露了出来，这既是优点也是缺点。
