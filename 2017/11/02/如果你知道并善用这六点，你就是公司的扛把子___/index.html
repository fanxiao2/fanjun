如果你善用者这六点，公司的扛把子非你莫属

发表于2017-11-02更新于2020-09-19字数统计707阅读时长3分

正如之前一篇文章我们所看到的其实编程并没有多难。比较难的部分实际上是先要得到一个程序。计算机是比较务实的，你必须先告诉它们要做什么，直至最后的细节。编写大型程序是一项非常艰巨的挑战，如果没有系统的方法实践，几乎是不可能完成的。

为了方便顺畅的编写程序我们通常会将创建程序的过程分为几个阶段，并依据每个阶段中产生的信息来进行分工工作。

1、分析问题

首先确定我们要解决的问题是什么。尝试竟可能的分析透彻（多进行了解）“知己知彼”，只有分析透彻才能知道要解决什么样的问题，否则一定不要开始去解决它。没搞清楚问题之前就去“实践”一定会给后续的开发带来很多问题。

2、确定规格说明

准确的描述程序将要怎么执行（程序将做什么）。此时，不必担心程序“怎么做”（代码怎么写），而是要确定它“做什么”。对于简单的程序，这包括仔细描述程序的输入和输出是什么以及他们的相互关系。

3、创建设计

这个步骤就是规划程序的总体结构。白话说就是描述程序怎么做的地方做成什么样子。主要任务是设计算法来满足规格说明。

4、实现设计

将设计翻译成计算机语言并放入计算机中来运行（简称编程）。在后面的文章中，我们将算法实现为 Python 程序。

5、测试/调试程序

试用你的程序，看看它是否按预期工作。如果有任何错误（通常称为“缺陷”），那么你应该回去修复它们。定位和修复错误的过程称为“调试”程序。在调试阶段，你的目标是找到错误，所以应该尝试你能想到的“打破”程序的一切可能。记住这句老格言：“没有什么能防住人犯傻，因为傻子太聪明了。”

6、维护程序

继续根据用户的需求开发该程序。大多数程序从来没有真正完成，它们在多年的使用中不断演进。
